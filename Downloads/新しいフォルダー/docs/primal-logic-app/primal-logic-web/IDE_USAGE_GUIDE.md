# IDE使い分けガイド - Cursor / AntiGravity / Claude Code

## 各IDEの特徴と使い分け

### 1. Cursor（現在使用中）

**特徴:**
- VS Codeベースで、既存のVS Codeユーザーは違和感なく移行可能
- AI統合（エディター内でAIチャット、コード補完、リファクタリング）
- インライン編集（コードを選択して自然言語で指示）
- コードベースのインデックス化（プロジェクト全体を考慮した提案）
- `.cursorrules`でAIの動作をカスタマイズ可能

**適した用途:**
- メイン開発（複雑な実装、AI機能の開発）
- コード補完やリファクタリング
- エラー修正やデバッグ
- 既存コードの理解と修正

**課金:**
- 有料プランあり（無料プランもあり）

---

### 2. AntiGravity

**特徴:**
- エージェント・ファースト（AIエージェントが主体的にコード生成・修正）
- ブラウザ上で動作（オフライン不可）
- 無料プランあり
- **自動デプロイ機能**あり（Netlify等への自動デプロイ）

**適した用途:**
- 新規機能のプロトタイプ作成
- UI調整や軽微な修正
- 自動デプロイが必要な場合
- コストを抑えたい場合

**課金:**
- 無料プランあり

---

### 3. Claude Code（Anthropic）

**特徴:**
- Claude AIを統合したコードエディター
- 高品質なコード生成と説明
- 課金が必要（無料プランは制限あり）

**適した用途:**
- 複雑なロジックの設計・実装
- コードレビューや改善提案
- ドキュメント作成
- アーキテクチャ設計

**課金:**
- 課金が必要（無料プランは制限あり）

---

## 現在のプロジェクトでの推奨

### 推奨: Cursorメイン + AntiGravity補助

**理由:**
1. **Cursorの`.cursorrules`が充実**: 既に詳細なルールが設定されており、AIの動作が最適化されている
2. **AntiGravityの自動デプロイ**: Netlifyへの自動デプロイで効率化
3. **コスト効率**: AntiGravityの無料プランで自動デプロイを活用
4. **Obsidian（第二の脳）の共有**: AntiGravityで`second-brain`フォルダを参照・編集可能

**使い分け:**
- **Cursor**: 機能実装、バグ修正、コード補完、リファクタリング、具体的な実装指示
- **AntiGravity**: 複数Agentで複数のタスクを並行実行、UI調整（抽象的な指示）、自動デプロイ、軽微な修正、Obsidianノートの参照・編集

**Obsidian（第二の脳）の共有:**
- AntiGravityでプロジェクトフォルダ全体（`docs`フォルダ）を開く
- `second-brain`フォルダを参照・編集可能
- 動的に共有（編集すると即座に反映）

## Claude Codeについて

### 課金していない場合

**使えるか:**
- 無料プランは制限がある（リクエスト数、機能制限など）
- 基本的なコード生成は可能だが、制限あり

**使うべきか:**
- **現時点では不要**: CursorとAntiGravityで十分
- **使うべき場合**: 
  - 複雑なアーキテクチャ設計が必要な時
  - 高品質なコードレビューが必要な時
  - 課金しても良い場合

## 実践的な使い分け

### 日常的な開発
1. **Cursorで開発** → コード実装、バグ修正
2. **Gitにコミット・プッシュ**
3. **AntiGravityで自動デプロイ** → Netlifyへの自動デプロイ

### 新機能の実装
1. **Cursorで実装** → メイン開発
2. **AntiGravityでUI調整** → 軽微な修正
3. **Cursorで最終確認** → コードレビュー

### デプロイ
1. **AntiGravityの自動デプロイ** → 推奨
2. **またはCursorから手動デプロイ** → 必要に応じて

---

## 注意点

1. **同時に開発サーバーを起動しない**: ポート競合を避ける
2. **Git操作は一つのIDEから**: 両方で同時に操作しない
3. **環境変数は両方で同じ値**: `.env`ファイルを共有
4. **`.cursorrules`は両方で有効**: AntiGravityも`.cursorrules`を読み込む
