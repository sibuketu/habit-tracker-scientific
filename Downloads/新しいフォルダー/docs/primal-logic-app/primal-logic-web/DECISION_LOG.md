# Decision Log - Primal Logic

> **目的**: 重要な決定事項と、その理由（Why）を記録する。
> **ルール**: 決定した内容だけでなく、「なぜその選択をしたか」を必ず記録する。

---

## 2026-01-11: If-Then機能の削除決定

### 決定内容
**If-Then機能は実装しない。**

### 理由（Why）
1. **ややこしくなる**: If-Then機能はHabitica/Streaksのような汎用習慣化アプリの機能。カーニボアアプリには不要な複雑性を追加する。
2. **Todoで十分**: 「外食時の対応」「糖質摂取時の対応」は、Todoリストやチェックリストで十分対応可能。
3. **カーニボア哲学に反する**: カーニボアは「シンプル」が哲学。機能を増やすことで本質から離れる。

### 参考
- `FEATURE_INTENTS.md` の「If-Thenルール機能」セクション
- 習慣化アプリ（Habitica、Streaks）のIf-Then機能を参考にしていたが、削除決定。

## 2026-01-11: Phase概念の削除決定

### 決定内容
**Phase 1、Phase 2、Phase 3などの概念を使用しない。**

### 理由（Why）
1. **CLAUDE.mdのルールと矛盾**: CLAUDE.md（105行）に「Phase概念なし：リリース時に自信を持って機能を提供する。Phase 1、Phase 2、Phase 3などの概念は使わない」と明記されている。
2. **実装方針の混乱**: Phaseで分割すると「Phase 1が終わるまでPhase 2を実装しない」という誤解を生む。実際は並行実装が可能。
3. **大量実装方針に反する**: リリース前に全部実装して見せる方針。Phaseで分割すると段階的リリースを示唆してしまう。

### 変更内容
- `Carnivore_Logic_Master_Spec.md`: 「Phase 1/2/3」→「Implementation Status」に変更
- `GEMINI_FEATURES_TO_IMPLEMENT.md`: 「Phase 2/3/4/5」を削除
