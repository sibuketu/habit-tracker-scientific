# Cursor チャット履歴の復活方法

## 問題: Reopenしたらチャットが消えた

CursorをReopenした際にチャット履歴が消えてしまう問題の対処法です。

## 解決方法

### 1. 自動再接続を待つ
Cursorは自動的に再接続を試みます。数秒待ってみてください。

### 2. 手動で再接続
- **Ctrl + R** (Windows) / **Cmd + R** (Mac): ページをリロード
- **F5**: ブラウザをリロード

### 3. エージェント管理画面から確認
- 右側のエージェント管理画面（画像参照）で、過去のエージェントセッションを確認
- 「Chat name generation...」や「∞ 認証機能の選択肢AとAI...」などのエージェントをクリックして、過去の会話を確認

### 4. エージェントの再作成
- 「New Agent」ボタンをクリックして新しいエージェントを作成
- 過去の会話内容を手動で再入力

### 5. チャット履歴の保存（推奨）
- 重要な会話は、定期的にコピーしてObsidianやテキストファイルに保存
- エージェント管理画面で過去のエージェントセッションを確認し、必要に応じて再開

## エージェント管理画面について

画像に表示されているエージェント管理画面では：
- **Search Agents...**: エージェントを検索
- **New Agent**: 新しいエージェントを作成
- **Agents**: 既存のエージェントリスト
  - 各エージェントには最終更新時刻が表示される
  - エージェントをクリックして過去の会話を確認可能

## 注意事項

- Cursorのチャット履歴は、エージェントセッションごとに管理される
- Reopenした際に、エージェントセッションが失われる場合がある
- 重要な会話は、定期的にバックアップを取ることを推奨

## WorkTreeについて

WorkTreeは、同じリポジトリ内で複数のブランチを同時に作業する際に使用します。通常の開発作業ではLocalを使用してください。

WorkTreeが必要な場合：
- 同じリポジトリ内で複数のブランチを同時に作業する場合
- マルチエージェント機能を使用する場合
- エージェントごとに独立した作業環境を確保したい場合

詳細は`CURSOR_LOCAL_VS_WORKTREE.md`を参照してください。
