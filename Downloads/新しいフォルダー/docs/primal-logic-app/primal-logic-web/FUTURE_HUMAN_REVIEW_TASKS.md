# 将来的に人間の確認・作業が必要な作業

> 作成日: 2025-12-29
> 目的: AIが手順を説明すれば人間でも対応可能な作業、または専門家の確認が必要な作業を記録
> 
> **注意**: 「人間チェックが必要」= AIができない、という意味ではありません。
> AIが手順を説明すれば人間でも対応可能な作業も含みます。

---

## セキュリティが重要な機能

### 認証・決済
- [ ] 決済機能の実装（Gift機能の決済部分）
  - **AIが手順を説明すれば人間でも対応可能**
  - Stripe/RevenueCatの統合（公式ドキュメントに従えば実装可能）
  - 決済フローのセキュリティレビュー（実装後の確認）
  - 決済情報の適切な管理（Stripeのベストプラクティスに従う）
- [ ] 認証機能の強化（リリース前に対応が必要か判断）
  - **現状**: 認証機能は実装済みだが、オプション（Local First方針）
  - **リリース前**: 認証を必須にするか、オプションのままにするか判断
  - セッション管理のセキュリティレビュー（実装後の確認）
  - パスワードリセット機能のセキュリティレビュー（実装後の確認）
  - ソーシャルログインの実装（Google/Apple）（公式ドキュメントに従えば実装可能）

### データ保護
- [ ] 個人情報の暗号化
  - 機密情報の暗号化方法のレビュー
  - データ転送時の暗号化確認
- [ ] GDPR対応の最終確認
  - データエクスポート機能の法的レビュー
  - データ削除機能の法的レビュー

---

## パフォーマンスが重要な機能

### 大量データの処理
- [ ] 大量の履歴データの処理
  - パフォーマンステスト（負荷テスト）
  - ボトルネックの特定と最適化
- [ ] リアルタイム通信の実装
  - WebSocketの実装とパフォーマンステスト
  - リアルタイム更新の最適化

## 複雑なデータベース設計

### 新規テーブルの追加
- [ ] コミュニティ機能のデータベース設計
  - テーブル設計のレビュー
  - インデックスの最適化
  - クエリのパフォーマンス確認
- [ ] 大規模なデータマイグレーション
  - データ移行スクリプトのレビュー
  - データ整合性の確認

---

## 専門知識が必要な機能

### 法的要件
- [ ] 利用規約・プライバシーポリシーの法的レビュー
  - **AIがテンプレートを生成し、人間が確認・修正**
  - 弁護士による最終確認（法的リスクを最小化するため）
  - 特定商取引法への準拠確認
- [ ] 決済関連の法的要件
  - **AIが手順を説明すれば人間でも対応可能**
  - 返金ポリシーの法的レビュー（Stripeの返金ポリシーに従う）
  - 領収書発行の法的要件確認（Stripeが自動発行）

### 医療・健康情報
- [ ] 健康情報の取り扱い
  - 医療アドバイスではないことの明確化
  - 免責事項の法的レビュー

---

## 注意事項

- これらの作業は、**実際に必要になった時点で判断**する
- AIだけで実装できる部分は先に実装し、人間チェックが必要な部分だけを後で対応する
- セキュリティや法的要件に関わる作業は、専門家（エンジニア、弁護士）に相談する
