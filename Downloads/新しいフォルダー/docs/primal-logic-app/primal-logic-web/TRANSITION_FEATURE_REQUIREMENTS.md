# 移行機能の要件定義

## 目的
カーニボアダイエット開始から30日間の移行期間をサポートし、症状に応じた対処法を提供する機能。

## 現状の実装状況

### 既存実装
- `transitionGuide.ts`: 移行期間の進捗計算と症状データ
- `TransitionBanner.tsx`: 移行期間バナー（進捗表示）
- `TransitionGuideModal.tsx`: 移行ガイドモーダル
- 症状データ: 5種類（頭痛、こむら返り、疲労感、便秘、下痢）

### 不足している機能
1. 症状の自動検出・提案
2. 症状に応じた栄養素目標値の自動調整
3. 移行期間中の日次チェックイン
4. 症状の記録・追跡
5. より詳細な症状データ（現状5種類のみ）

## 要件定義（調査が必要な項目）

### 1. 症状の種類と頻度
- **調査項目**: カーニボア移行期間中に発生する症状の種類と頻度
- **情報源**: Dr. Ken Berry、Dr. Shawn Baker、Dr. Paul Saladino、Dr. Anthony Chaffee、Dr. Bart Kay、Dr. Paul Masonの情報
- **既知の症状**:
  - 頭痛（電解質不足）
  - こむら返り（マグネシウム不足）
  - 疲労感（エネルギー不足、電解質不足）
  - 便秘（マグネシウム・水分不足）
  - 下痢（脂質過多）
  - その他（調査が必要）

### 2. 症状の対処法
- **調査項目**: 各症状に対する具体的な対処法（栄養素調整、食品推奨）
- **既知の対処法**:
  - 頭痛: ナトリウム+2000mg
  - こむら返り: マグネシウム+200mg
  - 疲労感: 脂質+50g、ナトリウム+1000mg
  - 便秘: マグネシウム+200mg、水分+1L
  - 下痢: 脂質-30g

### 3. 移行期間の期間設定
- **現状**: 30日固定
- **調査項目**: 30日が適切か、個人差を考慮すべきか
- **考慮事項**: 
  - 既に何日か経過している前提か、開始日から30日か
  - 個人差（体質、健康状態、移行前の食事）を考慮すべきか

### 4. UI/UX要件
- **現状**: バナー表示のみ
- **必要な機能**:
  - 症状の記録画面
  - 症状に応じた栄養素目標値の自動調整
  - 日次チェックイン機能
  - 移行期間の進捗可視化

### 5. 集客・マーケティング要件
- **目的**: SNS集客で「アプリで移行をサポート」を訴求
- **必要な機能**:
  - 移行期間のサポート機能が目立つUI
  - 移行期間中のユーザー体験の向上
  - 移行期間終了後の成功体験の可視化

## 次のステップ

1. **調査**: カーニボア専門家の情報を収集し、症状と対処法を整理
2. **要件定義**: 調査結果を基に詳細な要件を定義
3. **実装**: 要件に基づいて機能を実装
