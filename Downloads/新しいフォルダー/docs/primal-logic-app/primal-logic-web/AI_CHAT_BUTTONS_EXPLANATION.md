# AIチャットボタンの違いと用途

## 2つのボタンの違い

### 💬 ボタン（吹き出しUIに変更）
- **用途**: モーダルUIから吹き出しUIに切り替える
- **特徴**:
  - 小さなウィンドウで表示される
  - ドラッグして移動可能
  - リサイズ可能（4つの角から）
  - **他の画面を見ながら使える**（マルチタスク向け）
  - 画面の一部に表示されるため、アプリの他の機能と並行して使用可能

### ⛶ ボタン（全画面表示）
- **用途**: モーダルUIを全画面表示に切り替える
- **特徴**:
  - 画面全体に表示される
  - **AIチャットに完全に集中できる**（集中モード）
  - 他のUI要素（ナビゲーション、ホーム画面など）が非表示になる
  - 長文の入力や深い対話に最適
  - もう一度押すと通常サイズに戻る

## 使い分けの目安

### 💬 吹き出しUIを使う場合
- アプリの他の機能（栄養ゲージ、履歴など）を見ながらチャットしたい
- 短い質問や確認をしたい
- マルチタスクで作業したい

### ⛶ 全画面表示を使う場合
- 長文の質問や相談をしたい
- AIとの深い対話をしたい
- 他の要素に邪魔されずに集中したい
- 複雑な問題について詳しく相談したい

## Geminiでの要件定義との関連

`CARNIVOS_GEMINI_CONVERSATION_SUMMARY.md`の「Phase 2: AI対話メインのUI構築」で以下の要件が定義されています：

> **AI対話UI**: ユーザーが「腹痛がする」と言えば、AIが直近のログ（日光、脂質、誠実さスコア）から原因を推論

全画面表示機能は、この「AI対話メインのUI構築」の要件を満たすための機能です：

1. **集中モード**: 他のUI要素を非表示にして、AIとの対話に集中できる
2. **深い対話**: 長文の質問や複雑な問題について、邪魔されずに対話できる
3. **推論機能**: AIがユーザーのログから原因を推論する際、集中して対話できる環境を提供

## 実装の意図

全画面表示機能は、以下の意図で実装されています：

- **ユーザーエクスペリエンス**: ユーザーがAIとの対話に集中できる環境を提供
- **機能の使い分け**: 用途に応じてUIを切り替えられる柔軟性
- **Gemini要件の実現**: 「AI対話メインのUI構築」の要件を満たす
