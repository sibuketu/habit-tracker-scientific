## コピペ用メッセージ

```
はじめまして。龍谷大学一年の須佐見 隼人です。

CarnivoreGuidというカーニボアダイエット管理アプリをCursorなどのAIツールで開発しました。主要機能を実装済みで、デモURLはこちら: https://scintillating-dodol-e8fcf2.netlify.app/

【カーニボアダイエットとは】
肉、魚、卵、内臓などの動物性食品のみを摂取し、植物性食品を完全に除去する食事法です。植物毒を避け、血糖値の安定、満腹感の持続、腸の健康改善を目的としています。世界的に急成長している健康ムーブメントであり、科学的根拠に基づいた実践者が増えています。

【CarnivoreGuidとは】
カーニボアダイエット管理アプリです。このアプリは単なる「記録ツール」ではなく、「カーニボアという急成長市場における、唯一の論理的プラットフォーム」を目指しています。

【主な機能（実装済み）】

1. 栄養素追跡システム
- P:F Ratio（タンパク質:脂質比率）: ホーム画面の最上部にゲージとして常時表示
- 栄養素ゲージ: タンパク質、脂質、ビタミン、ミネラルなどを追跡
  - Vitamin C（動的必要量対応: 糖質<20gの場合は必要量が大幅に減少）
  - Vitamin K、Iron、Sodium（5000-8000mg）、Fiber（ゼロ目標）、Net Carbs（ゼロ目標）
  - Zinc、Magnesium、Potassium、Vitamin D、Vitamin A、Vitamin B12、Choline など
- 動的栄養素目標値: ユーザープロファイル（100項目以上の質問）に基づいて自動調整
  - 年齢、性別、活動量、ストレス、睡眠、運動、甲状腺機能、日光暴露、消化器系の問題、炎症、メンタルヘルス、サプリメント、アルコール、カフェインなど
  - 各設定項目が栄養素目標値に与える影響を「タンパク質+10g」のように表示
  - プレビュー機能で現在の設定による栄養素目標値の変動を確認可能

2. AIチャット（カーニボア専門家の情報を統合）
- 情報源: Carnivore医者（Dr. Ken Berry、Dr. Shawn Baker、Dr. Paul Saladino、Dr. Anthony Chaffee、Dr. Bart Kay、Dr. Paul Masonなど）
- 機能: 
  - 嘘情報を事前にはじいている
  - カーニボア専門家のクローンのようなAI
  - 論証スタイル（Validate → Deconstruct → Reframe → Cite）を完全に模倣
  - 症状の対処法提案
- 技術: Gemini API（gemini-2.5-flash）を使用
- コンテキストエンジニアリング: 構造化プロンプト（XMLタグ使用）、Chain of Thought (CoT) の言語分離、Chain of Verification (CoVe) による自己検証、RAGと引用機能でハルシネーション防止

3. 写真解析とAI Conciergeインタビュー
- 機能: 食べたものを写真で記録
- AI Conciergeインタビュー: 写真解析後に、不足情報を聞き出す（Macのサイドメニューのようなチェックボックス形式）
  - 塩は振りましたか？（デフォルト: ぬちまーす）
  - 脂身は残しましたか？（トリミング、0-100%の範囲で入力可能）
  - 調理方法は？
  - バター/牛脂の確認（推測される食品についてもフォローアップ）
- 技術: Gemini API（画像解析機能）を使用

4. 「いつもの（My Foods）」タブ
- 機能: 履歴を活用したワンタップ入力
- 目的: キーボード入力を排除し、3秒で完了
- 実装: 履歴から最も頻繁に追加された食品を上位10件表示

5. Interactive Butcher（食品選択UI）
- 機能: 牛・豚・鶏の解剖図（SVG）を使用した食品選択
- 特徴: グラム入力は「微調整」のみ。基本は「スライダー」や「見た目プリセット（ステーキ1枚など）」
- UI: タップで部位を選択、スライダーで量を調整

6. Argument Cards（論破カード）
- 機能: 栄養素メトリクスをタップすると、Argument Cardが表示される
- 内容: 
  - 3-tier表示（Level 1/2/3）
  - Verdict表示（trash/sufficient/precious/safe）
  - 栄養素別Argument Card
  - カーニボア専門家の論理を引用
- 目的: 現代の栄養ノイズに対する「盾」と論理的自衛の「武器」

7. Recovery Protocol（回復プロトコル）
- 機能: 違反食品（trash/fiber/carbs）を検出すると、自動的にRecovery Protocolを生成
- 内容: 
  - ファスティング時間計算
  - 食事推奨事項生成
  - 明日のログに追加機能

8. ユーザープロファイル設定
- 機能: 100項目以上の質問項目で、栄養素目標値を動的に調整
- カテゴリ: 
  - 検査不要（自己申告で分かる）: 年齢、性別、活動量、ストレス、睡眠、運動、甲状腺機能、日光暴露、消化器系の問題、炎症、メンタルヘルス、サプリメント、アルコール、カフェインなど
  - 検査必要（測定が必要）: 血液検査値など

9. 履歴画面
- 機能: 日次ログ履歴を表示
- 特徴: 
  - アコーディオン形式の詳細表示
  - 削除機能
  - Argument Card統合
  - Recovery Protocol表示

10. Logic Armor（理論武装）
- 機能: カーニボアダイエットに関する知識ベース
- 内容: カーニボア専門家の情報、論理、研究データ

11. 設定画面
- 機能: アプリ設定（文字サイズ、表示設定など）
- 表示設定: 
  - 知識ベースを表示（ON/OFF）
  - 栄養素プレビューを表示（ON/OFF）

12. 音声入力
- 機能: Web Speech APIを使用した音声入力
- 用途: 食品追加時の入力補助

【カーニボアロジック（独自の論理）】

動的栄養素目標値:
- Vitamin C: 糖質<20gの場合は必要量が大幅に減少（壊血病を防ぐために必要な最小量は約10mg。肉食のみでも1日約30mgを摂取可能なため、肉で十分足りる）
- Vitamin K: 動的必要量対応
- Iron: 動的必要量対応（女性の月経、閉経、妊娠、授乳などに応じて調整）
- Sodium: 5000-8000mg（低インスリン状態では腎臓からのナトリウム排出が加速するため、電解質維持と「ケトフル」予防に高用量が必要）
- Magnesium: 600mg（現代の土壌枯渇により肉のみでは不足しがちであり、こむら返りや便秘を防ぐために積極的な摂取またはサプリメントが必要）
- Potassium: 4500mg（ナトリウム摂取増に対応する細胞内バランス維持のため、肉汁（ミオグロビン）を捨てずに摂取することで確保する）

バイオアベイラビリティロジック:
- 植物性栄養素: ペナルティを適用（例: 植物性タンパク質はDIAASスコアで評価、植物性鉄は吸収率が低い）
- 動物性栄養素: 優先（例: ヘム鉄は吸収率が高い）
```
