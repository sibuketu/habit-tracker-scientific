# 移行機能のおすすめ実装

## 優先順位（推奨順）

### 1. **症状の自動検出・提案機能**（最優先）
- **理由**: ユーザーが症状を記録する手間を減らし、AIが自動的に症状を検出して対処法を提案
- **実装内容**:
  - AIチャットに「今日頭痛がする」「こむら返りが起きた」と話しかける
  - AIが症状を検出し、対応する栄養素調整を自動提案
  - 栄養素目標値の自動調整（例：頭痛→ナトリウム+2000mg）
- **UI**: AIチャット内でTodoとして表示、実行ボタンで自動調整

### 2. **日次チェックイン機能**（高優先度）
- **理由**: 移行期間中の継続的なサポートと進捗追跡
- **実装内容**:
  - 移行期間中、毎日自動でチェックインを促す通知
  - 簡単な質問（「今日の調子は？」「症状はある？」）
  - AIが回答を解析し、必要に応じて栄養素調整を提案
- **UI**: バナーまたは通知で表示、AIチャットで回答

### 3. **症状の記録・追跡機能**（中優先度）
- **理由**: 症状のパターンを把握し、個人に最適化した対処法を提供
- **実装内容**:
  - 症状の記録画面（簡単な選択式）
  - 症状の履歴表示（グラフ化）
  - 症状パターンの分析（AIが自動分析）
- **UI**: InputScreenに統合、または専用画面

### 4. **より詳細な症状データ**（中優先度）
- **理由**: 現状5種類のみなので、より包括的なサポートが必要
- **実装内容**:
  - 症状データの拡充（調査が必要）
  - 各症状に対する詳細な対処法
  - 症状の組み合わせへの対応
- **調査項目**: カーニボア専門家の情報を収集

### 5. **移行期間の進捗可視化強化**（低優先度）
- **理由**: 現状のバナーで十分だが、より詳細な可視化でモチベーション向上
- **実装内容**:
  - 進捗グラフ（日次）
  - 症状の推移グラフ
  - 栄養素調整の履歴
- **UI**: 移行期間専用画面または既存画面に統合

## 実装方針

### Phase 1: AI音声入力による自動記録（最優先）
- InputScreenのデータをAI音声入力で自動入力
- 「今日の睡眠は95点、8時間、太陽光は30分、排泄は正常」と話しかける
- AIが構造化データに変換して自動入力

### Phase 2: 症状の自動検出・提案
- AIチャットで症状を話しかけると自動検出
- 栄養素目標値の自動調整を提案

### Phase 3: 日次チェックイン
- 移行期間中の自動チェックイン機能

## 推奨実装順序

1. **AI音声入力によるInputScreen自動記録**（今回実装）
2. **症状の自動検出・提案**（次回実装）
3. **日次チェックイン**（その次）
