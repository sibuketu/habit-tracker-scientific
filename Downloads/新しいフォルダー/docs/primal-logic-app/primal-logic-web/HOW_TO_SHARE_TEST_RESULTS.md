# テスト結果の共有方法（サルでもわかる）

> 作成日: 2026-01-03
> **Playwright Test Reportの結果を共有する方法**

---

## 🎯 方法1: HTMLレポートを開く（推奨・最も簡単）

### 手順

1. **エクスプローラーを開く**（Windowsキー+E）
2. **アドレスバーに以下を貼り付けてEnter:**
   ```
   C:\Users\susam\Downloads\新しいフォルダー\docs\primal-logic-app\primal-logic-web\playwright-report
   ```
3. **`index.html` をダブルクリック**
   - ブラウザでPlaywright Test Reportが開きます
4. **スクリーンショットを撮る**
   - Windowsキー + Shift + S（Snipping Tool）
   - または、PrintScreenキー
   - 失敗したテストの詳細部分をスクリーンショットで撮影
5. **スクリーンショットをチャットに貼り付け**

---

## 🎯 方法2: エラーメッセージをコピー&ペースト

### 手順

1. **Playwright Test Reportを開く**（方法1を参照）
2. **失敗したテストをクリック**
   - 左側のリストから失敗したテストを選択
3. **エラーメッセージをコピー**
   - エラーメッセージ部分を選択してCtrl+C
4. **チャットに貼り付け**
   - Ctrl+Vで貼り付け

---

## 🎯 方法3: ターミナルの出力を共有

### 手順

1. **ターミナルでテストを実行**
   ```powershell
   cd "C:\Users\susam\Downloads\新しいフォルダー\docs\primal-logic-app\primal-logic-web"
   npm test
   ```
2. **エラーメッセージをコピー**
   - ターミナルのエラーメッセージ部分を選択してCtrl+C
3. **チャットに貼り付け**
   - Ctrl+Vで貼り付け

---

## 🎯 方法4: 失敗したテストの詳細をテキストで共有

### 手順

1. **Playwright Test Reportを開く**（方法1を参照）
2. **失敗したテストをクリック**
3. **以下の情報をメモ**
   - テスト名（例: "6: 栄養素ゲージ表示"）
   - エラーメッセージ（例: "Element not found"）
   - スクリーンショット（あれば）
4. **チャットに貼り付け**

---

## 📋 共有すべき情報

### 最低限必要な情報

1. **失敗したテストの数**
   - 例: "140個のテストが失敗しました"
2. **失敗したテストの名前**
   - 例: "6: 栄養素ゲージ表示"
3. **エラーメッセージ**
   - 例: "Element not found: getByText(/ビタミンC|Vitamin C/i)"

### より詳しい情報（推奨）

1. **スクリーンショット**
   - 失敗したテストの詳細画面
   - エラーメッセージが表示されている部分
2. **複数の失敗したテストの例**
   - 3-5個の失敗したテストの詳細
3. **成功したテストの数**
   - 例: "58個のテストが成功しました"

---

## 🚀 クイック共有（推奨）

### 最も簡単な方法

1. **Playwright Test Reportを開く**
   - `playwright-report/index.html` をダブルクリック
2. **失敗したテストを3-5個選ぶ**
3. **各テストをクリックして、エラーメッセージをスクリーンショットで撮る**
4. **スクリーンショットをチャットに貼り付け**

これで十分です！

---

## 📝 補足

- **HTMLレポートの場所**: `primal-logic-app/primal-logic-web/playwright-report/index.html`
- **スクリーンショットの場所**: `primal-logic-app/primal-logic-web/test-results/`（失敗したテストのスクリーンショットが保存されている）
- **レポートを再生成**: `npm test` を実行すると、最新のレポートが生成されます
